关于我玩Phigros的历程

2022-04-15

一转眼我玩phigros（以下简称“phi”）快半年了，当初怎么入坑的也不记得了，刚入坑第一想法：好简单，好快乐。入坑差不多一个多月内的时间只打HD难度的，EZ不屑打，IN打不过。之前最喜欢的歌就是Sultan Rage，然后就是各种疯狂肝，甚至离谱到闭上眼睛都能打前面那一段……

那段时间是什么都不记的，什么note键全都不知道是啥，就记红键蓝键黄键。然后突发奇想去打了ENERGY SYNERGY MATRIX，被虐的很惨，这个故事告诉我们不要越级打歌。啊然后我的肝就放在了这两首歌上面，每天这个一遍那个一遍，结果就是一个上了蓝V（95W），一个卡在A(91w)再起不能。

接着就是打主线，你知道这主线我是怎么过的吗！（拍桌）。说实话，对于我这样一个只想打自己喜欢的歌的“佛系玩家”，要把完全不喜欢的歌都打一遍还得A或S才能解锁，我的内心是崩溃的。啊当然每首歌的创作者都很厉害，歌也很好，是鄙人欣赏不来，抱拳，告辞（求生欲满分）。

好像是在第五章，隐藏歌开始了。我第一次碰到的时候一脸懵逼，没想起来查攻略，傻呵呵的把整个phi都翻了一遍也没找到。一查攻略，哦要A以上解锁隐藏啊，然后我就死抠着Spasmodic的HD难度打了N遍，再一看，我去打EZ都能解，不早说？！自此我就开始了我的……应该说fw之录吧？因为从我知道这些隐藏歌打EZ也能解锁之后，我就没怎么打过HD了。

第六章最草率，为了赶紧解锁魔王曲就打的飞快，全打的EZ。然后就是跟着攻略走，看见攻略说有一个下落比较缓慢的蓝键不能按的时候我还有点紧张，因为我怕按错，再者就是真的不想打第二遍了。

第七章我就觉得挺麻烦的，因为全部曲子打完一遍都是A以上之后再重新打第一首就会变成隐藏曲，最最关键的是，隐藏曲他要全连啊！！！就因为这个全连，我重新打了两遍才过……当时我还觉得特激动来着，毕竟是最初版本的phi打魔王曲，截了挺多张图，最后都因为内存不够被删了。

然后我就开始“养老”，打魔王曲的HD，结果怎么都打不到S……91w是我的极限了。

这个时候，MARENOL出现在了我的视线里。我挺喜欢这种歌的，因为听着就有一种“啊我真帅”“啊我就是末日唯一幸存者”这种沙雕的感觉……这个比喻可能不太好，但是我也想不到其他的比喻了23333。我就一直在打这首，因为是手机所以操作不是很方便，而且我这小米手机老是打一半突然卡一下，等再连上的时候就只有一句恶魔低语“你 线 白 了”。

而且这还是看运气的，唉我真无语。

我在B站就刷到了996，挺想尝试一下的，但是怎么搜都给我显示996只要更新到最新版本就有，我还以为是又要打歌呢。我的phi是浏览器搜出来下的，因为应用商店没有，所以网上搜出来的那些更新方法对我似乎都没啥用。我看有人说卸掉重新装就能更到最新版本，但是我不敢啊。有了先前在iPad上删掉再下啥也没了的经验，我是真的不敢删了。于是我就在iPad上又下了回来，就单纯打996，手机上就不管了。

结果事情出现了转机，我当时就是想在手机上也有996打，但又不想卸载，就寻思着下个taptap再下一个phi。我在taptap上下载了phi之后没在桌面上发现新的phi，回去一看哦豁原来人家是直接在taptap里面玩的。激动人心的时刻到来了，我登上去发现是自己的号！我怀着激动的心情退出了taptap去看手机上的phi，哇它真的更了！！！没有人知道我当时有多开心，狂喜了简直。

说完了过去，来说说我现在都在干啥。首先就是前面提到过的MARENOL，手机卡95w了，iPad到是上了99w。996也是，iPad92w手机91w卡死。然后就是Another Round，手机有98w，iPad因为MB不够了所以没解锁。还有就是rks，写这篇的时候我的rks才9.62……我朋友都疑惑为什么这么低，我去浏览器一搜，他告诉我rks跟准度和你打的歌的多少计算……对于我这种只打爱听的歌的人非常不友好！！！（梅开二度）放几张图



该讲的到这里就讲完了，一篇基本都是流水账，我果然没有写作文的天赋啊唉

上网课时的一些奇闻轶事

2022-04-15

4月11日，广州全市中小学暂停线下教学！体育中考暂停！正当同学们欢呼不用上课时，老师一条信息把同学们打回原形：

同学们！虽然因为疫情我们不能一起快乐的线下学习，但是我们可以在网络上继续我们的愉快学习之旅！我们明天，线上见！

这一条短信威力可不小，我们全班同学瞬间不敢说话，但总还是有那么一丝窃喜：不用线下上课嘛，可以摸鱼哦，总比线下好啊！
于是，第二天下午信息课时，大家十分惬意，躺在床上、沙发，坐在椅子上，看着老师自己一个人演哑剧，自己却在开分屏或小窗舒舒服服的玩游戏甚至淘宝购物呢！
还有一种虽无聊但能打发时间的玩法，给老师的直播点赞！听起来无聊，但你要是真正上网课也只能玩这个了。而且，这种人多就好玩，玩了还能练你的手速。如果不想费手，你可以开个连点器，时时刻刻为老师点赞！

星期二，下午有一节美术课，这是我们班男孩子们的噩梦（不是全部），有些人（比如本人）就开始捉弄老师：

还有课堂上的默契交流：

当然，这只是低级玩法，但只要老师开的是会议……

就会有一堆奇才出现！

在各大办公平台上（如钉钉、腾讯会议等诸多恶魔神器），有一种功能是“虚拟背景”。这种功能的原意是让参会人的重点放在人脸上，不会被其他事物分心（大概是这样）。但这却被神通广大的同学们玩坏了。对此，B站公众号也写了篇文章：哔哩哔哩·你永远不知道，00后上网课能整出多少活

在我们班的会议上，仅有我一个人，打开了摄像头，就是为了尝试这个有趣的功能。
为此，我上网搜了几个沙雕的背景（图源来自知乎），放上来给你们看看：


我依次试了试，效果很尽人意。

太多啦，我就不一一展示啦。

虽然这些都非常好玩，但快乐总是很短暂的。
今天，4月15日，新闻报道称下周一、二会逐步恢复线下教学，我们这些在线上搞怪的学生们出来总还是要还的。最终还是会回到熟悉的校园生活。

所以让我们珍惜这段珍稀的时间，一起颓废……啊不……一起学习起来吧 (doge)！

杂谈·关于SCP基金会被禁这档子事

2022-03-26

今日我等若冷眼旁观，他日祸临己身，则无人为我摇旗呐喊。

鲁迅

关于 SCP 基金会

以下摘自萌娘百科

SCP基金会（Special Containment Procedures Foundation）是一个虚构的全球性超自然事物调查管理组织。

使命宣言

于世界各地的暗处活动，基金会的目标是收容异常物品，个体及现象，而且其本身运作不受各个主要国家政府的司法管辖权，授权和委托的干扰。这些异常透过物理或心理危害对全球安全造成显著威胁。

基金会维持常态，从而使世界各地的平民得以生存并免受恐惧，不信或对个人信念的怀疑的影响，并从地外、异次元和外层空间的影响中维持人类的独立自主。

我们的任务有三方面：

  1. 控制
    基金会控制异常以防止它们落入平民或敌对组织手中，透过广泛观察及监测并采取行动尽早拦截它们。

  2. 收容
    基金会收容异常以防止它们的影响或效应散播，通过迁移，掩饰，或拆除它们或通过抑制或阻止公众传播它们的知识进行。

  3. 保护
    基金会保护人类免受异常的影响并保护异常本身直至它们被完全理解以及出现自它们的特性及行为上制定的新科学定理为止。如果异常被收容时被判定为过度危险，基金会或也会以将异常无效化或摧毁作为最后手段。

正文

2021的某一天，我上百度搜索’SCP基金会官网’时，发现出现的并不是像平常一样的官网结果，而是像下面这样的‘儿童邪典’字样。

我觉得奇怪，点进中国网那个链接去看，戳这。

文章很长，主要讲得就是各种SCPs怎么怎么污染儿童心灵，如何bl，sq，不仅仅是SCPs们的锅，还有类似 “女性做手术”的外科游戏，如何xx，sw。

确实，这些东西不适合非成年人观看，但是，为何要把SCP所有的东西都一并清除呢？他们说B站上的视频容易让未成年人看到，就全删了。为啥啊，B站明明有未成年模式，更何况家长不将手机给孩子玩、或者把B站等应用在给孩子前先“应用限时”啊。明明有解决办法，但不少家长依旧将SCP基金会举报了，而基金会也消失在我们的视线中。

起初家长们疯狂举报虹猫蓝兔，我没有说话，因为我不看虹猫蓝兔；接着他们举报喜羊羊，我没有说话，因为我不看喜羊羊；后来他们举报了十万个冷笑话，我没有说话，因为我不看十万个冷笑话；后来他们举报了心跳文学部，我没有说话，因为我不玩心跳文学部；如今他们冲SCP基金会而来，我却发现再也没有人为我说话了。

雷霆之怒萌娘百科评论

除了SCP基金会，还有《查理九世》、《超级飞侠》、《虹猫蓝兔》等等诸多的书籍和动画片都被“家长们”举报。甚至连《超级飞侠》也被封杀了，我的妹妹小时候很喜欢看这部动画片，因为可以了解更多国家的历史、文化，看看他乡的风景，但家长们却以崇洋媚外为由封禁了这部动画，我妹妹吵着要看，但却在各大视频网站消失了。

到了现在，情况缓和了许多，用必应搜索基金会可以看到官网，在爱奇艺上也可以搜到超级飞侠，甚至还能看到新出的11季。

虽然放松很多了，但并不能代表将来没有更多的动画、虚构物等被举报。在这，我希望各大家长们说：“为了孩子，这样做是正确的，但也不能反应过度，不能因为单纯的‘为孩子好’就将SCP彻底封杀，还有许多成年人热爱这个社区，为这些社区做贡献。”

给Twikoo加个邮箱提醒

2022-02-12

跳坑原因

我还真没想明白我为什么要这样做就一个白痴，不过也是快开学了，一开学我就没有时间折腾博客了，乘着精力旺盛再折腾下呗~

写在最前

在Twikoo的开源文档中(戳这里),你可以看到，使用Vercel部署的云函数，请配置国外的邮件服务商，关键词:国外。所以免费部署在Vercel上的小伙伴们折腾时不要跳坑，要配置国外的服务商，这里建议Zoho国际版和Gmail(Gmail没试过，大家可以自己摸索)。本文主要讲Zoho国际版。

开始！

  1. 打开Zoho的国际版登陆界面，登录详见下图。
  2. 登录后会进入邮箱配置页，你要先拥有自己的域名才能继续（毕竟是企业邮箱嘛。你会看到这样。
  3. 按照它的操作一步一步做，依靠我依稀的记忆写个Tips。

设置用户: 随便配置一个你喜欢的名字，我就简单写了个niko，我的企业邮就是niko@nikoblog.top
设置组: 这玩意没什么实质意义，可以跳过。
DNS映射: 按照他的要求增加TXT或MX的DNS记录，等会给你们张图看看我的DNSPOD。
电子邮件迁移: 无必要，如果你要从其他邮箱迁移过来也行，自己折腾吧。不负责任的态度

如果是腾讯云买的域名可以看我的DNS记录，下图。

网站Twikoo配置

打开你的网站随意一篇文章，下滑到Twikoo评论那，打开设置->输密码->进入配置管理->打开邮件通知菜单，看下图。

PS: SENDER_EMAIL是之前设置的邮箱，忘了的话见下图。

升级+优化的网易云歌单下载器

2022-01-29

嘿，大家好，今天复习了一下爬虫知识，自己想爬个网易云的歌单下来，备着以后禁网的时候听。但由于我懒癌晚期，懒得直接上浏览器上搜，搜着，发现没有一个合格的。
为什么？其实是因为那些爬虫是打开浏览器（即Selenium）、打开歌单的网址、把其中的歌曲名和链接拿下、依次用外链下载。
这个方法不好，原因是在网页版的网易云的无登陆状态是只能在歌单里看到十首歌，无法看全。就好比我歌单里有50首歌，而你的爬虫只能爬10首下来，没效果。

没办法，只能自己写了。就是套了个api嘛。源码如下。

只有进度条版本的。。。

import requests
import json
import jsonpath
import os
from tqdm import tqdm


choose = int(input("1.歌单 2.歌曲"))
if choose == 1:

    playlist_url = input('请输入歌单网址: >>>')
    playlist_id = int(playlist_url.strip().split('=')[-1])

    myjson = requests.get(
        'https://api.injahow.cn/meting/?type=playlist&id={}'.format(playlist_id)
    )
    data = json.loads(myjson.text)

    urls = []
    for i in data:
        url = jsonpath.jsonpath(i, '$..url')

        urls.append(url[0])

    # https://api.injahow.cn/meting/?server=netease&type=url&id=1344088470
    print('总共有{}个链接，开始下载……\n'.format(len(urls)))

    os.makedirs('.\\dmusic\\playlist\\{}\\'.format(playlist_id),exist_ok=True)

    for url in tqdm(urls, desc='下载中'):
        song_id = url.strip().split('=')[-1]
        with requests.get(url) as resp:
            with open(
                '.\\dmusic\\playlist\\{}\\'.format(playlist_id) + str(song_id) + '.mp3',
                mode='wb',
            ) as f:
                f.write(resp.content)

elif choose == 2:
    song_url = input('请输入歌曲网址 (q退出): >>>')
    while song_url != 'q':
        song_id = int(song_url.strip().split('=')[-1])
        myjson = requests.get(
            'https://api.injahow.cn/meting/?type=song&id={}'.format(song_id)
        )
        data = json.loads(myjson.text)

        os.makedirs('.\\dmusic\\song\\',exist_ok=True)

        with requests.get(url=jsonpath.jsonpath(data, '$..url')[0]) as resp:
            with open(
                '.\\dmusic\\song\\' + str(song_id) + '.mp3',
                mode='wb',
            ) as f:
                f.write(resp.content)
        print(str(song_id) + "下载成功")
        song_url = input('请输入歌曲网址 (q退出): >>>')

效果:

如果看到那些用ID命名的歌曲不惯呢，可以打开网易云音乐桌面版 –> 打开本地音乐 –> 选择目录 –> 把缓存糖果屋添加进去 –> 添加完后，点击匹配音乐 –> ALL DONE!

Python-Requests

2022-01-17

本教程使用Python语言，需提前安装Pip3 or Pip，例如Linux类的，请在命令行内输入:

sudo apt install python3-pip

安装

一条命令(临时换源):
Linux:

sudo pip install requests -i https://mirrors.aliyun.com/pypi/simple/

Windows:

pip install requests -i https://mirrors.aliyun.com/pypi/simple/

使用

基础{Get请求}

import requests
data = {
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/97.0.4692.71 Safari/537.36'
}  # 向服务器传递的数据
response = requests.get('https://bing.com',data=data).text # text函数:获取对应网址的源代码
print(response)

User-Agent就像是浏览器的身份证,如果是Requests默认的ua的话，对应服务器会拒绝你的爬虫请求，简而言之，拿不到数据，所以我们要获得浏览器的ua。

进阶 {Post请求}

  1. 带数据的 post
# -*- coding:utf-8 -*-
import requests
import json
 
host = "http://httpbin.org/"
endpoint = "post"
url = ''.join([host,endpoint])
data = {'key1':'value1','key2':'value2'}
 
r = requests.post(url,data=data)
#response = r.json()
print (r.text)
  1. 多文件上传
# -*- coding:utf-8 -*-
import requests
import json
 
host = "http://httpbin.org/"
endpoint = "post"
 
url = ''.join([host,endpoint])
#多文件上传
files = [
    ('file1',('test.txt',open('test.txt', 'rb'))),
    ('file2', ('test.png', open('test.png', 'rb')))
    ]
 
r = requests.post(url,files=files)
print (r.text)

冷门 {Put 请求}

import requests
import json
url_put = "http://127.0.0.1:8080/"

}
param = {

}
payload = json.dumps(param)
response_put = requests.put(url, data=payload, headers=headers_put)

项目实战: 梨视频爬虫

这个项目我以前做过，就直接复制过来啦，摆烂啦 （

# !/usr/bin/python3
# -*- coding:utf-8 -*-
"""
@author: 
@file: 梨视频爬虫.py
@time: 2021/7/11 21:48
@desc: 
"""
import requests

url = "https://www.pearvideo.com/video_1731260"
headers = {
    "user-agent": "Mozilla/5.0 (Windows NT 10.0; WOW64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/86.0.4240.198 Safari/537.36",
    "referer": url,
}
contId = url.split("_")[1]
videoStatusUrl = f"https://video.pearvideo.com/mp4/third/20210603/cont-{contId}-15316010-202041-hd.mp4"
# 'https://video.pearvideo.com/mp4/third/20210603/cont-1731260-15316010-202041-hd.mp4'

resp = requests.get(videoStatusUrl, headers=headers).content
with open("video.mp4", mode="wb") as file:
    file.write(resp)
  1. 第14行,referer的定义，因梨视频请求头中必须有防盗链，即字典中的referer，如果没有，则获取不到视频数据。
  2. 第16、17行，通过抓包得知，梨视频的视频直链即为一串固定地址加上视频ID号，只需进行字符串分割即可。
  3. 第20行，向服务器发起GET请求，注意,.content是获取网页的二进制数据。
